# WeatherSpark reference / data-source audit (2026-07-10)

## Reference URL

- https://zh.weatherspark.com/y/137431/中国、徐行镇的全年平均天气

## Access result

- OpenClaw's `web_fetch` refused the page because the local DNS/proxy resolves `zh.weatherspark.com` to a private/special-use address.
- Direct `curl`/`requests` reached CloudFront but returned a zero-byte 202 body in this environment, so the live HTML/script payload was not available for reliable inspection.
- The implementation therefore uses the user's screenshots for module structure and method thresholds, and uses the repository's existing public EPW data pipeline instead of attempting to copy WeatherSpark raw data.

## Data source used in WeatherWhisper

- Source files: OneBuilding / Climate.OneBuilding.org EPW and STAT files stored under `data/raw/`.
- Processed files: `data/processed/*.json` and `public/data/*.json` generated by `scripts/processor.py`.
- Data type: typical meteorological year / CSWD-style hourly weather files, including dry-bulb temperature, dew point, relative humidity, solar radiation, wind speed/direction, total/opaque sky cover, visibility and liquid precipitation.

## Implemented WeatherSpark-style modules

1. Average high / low / mean temperature, plus apparent temperature.
2. Hourly-by-month average temperature heatmap.
3. Monthly cloudiness categories: clear, mostly clear, partly cloudy, mostly cloudy, overcast.
4. Precipitation probability and monthly precipitation.
5. Dew-point humidity comfort levels and apparent temperature.
6. Wind speed and vector-averaged dominant wind direction.
7. Tourism score with method panel.

## Tourism score method encoded in this repository

- Analysis hours: 08:00-21:00 local time.
- Hourly comprehensive score:

```text
0.50 * apparent-temperature score
+ 0.25 * cloud score
+ 0.25 * precipitation score
```

- Tourism temperature thresholds from the screenshot: below 10°C = 0, 18°C = 9, 24°C = 10, 27°C = 9, 32°C and above = 1. Values between thresholds are linearly interpolated.
- Cloud score: clear sky = 10, mostly clear ≈ 9, overcast = 1, linear in between.
- Precipitation score: no precipitation = 10, trace precipitation ≈ 9, >=1 mm/hour = 0, linear in between.
- A beach/pool reference score is also calculated with the screenshot's warmer-temperature thresholds.

## Boundary

This project reproduces the analytical style and user-facing modules of WeatherSpark, not its proprietary chart images or raw data. If the owner later provides explicit access to a licensed data feed, `scripts/processor.py` can be adapted to ingest it while preserving the same public JSON schema.
